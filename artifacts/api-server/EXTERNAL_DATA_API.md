# External Data API Documentation

Use this document with third-party integrators who need to write/read the JSON cache.

## Base URL

- Production: `http://72.60.180.99/api`

---

## 1) Refresh Data (Overwrite)

### Endpoint

- `POST /external-data/refresh`

### Purpose

- Accepts JSON payload from client
- Replaces existing file content completely
- Server automatically adds `updatedAt` timestamp

### Headers

- `Content-Type: application/json`

### Request Body

- Any valid JSON is accepted:
  - Object (recommended), e.g. `{ "users": [...] }`
  - Array/string/number also accepted

### Behavior

- If body is an object:
  - saved as object + `updatedAt`
- If body is non-object:
  - saved as:
    - `data`: original payload
    - `updatedAt`: timestamp

### Success Response

- `200 OK`

```json
{
  "ok": true,
  "file": "third-party-data.json"
}
```

### Error Responses

- `400 Bad Request`: missing/invalid JSON body
- `500 Internal Server Error`: server write failure

### cURL Example

```bash
curl --location 'http://72.60.180.99/api/external-data/refresh' \
--header 'Content-Type: application/json' \
--data '{
  "users": [
    { "id": 1, "name": "Ali" },
    { "id": 2, "name": "Sara" }
  ],
  "source": "partner-system"
}'
```

---

## 2) Read Cached Data (Read-Only)

### Endpoint

- `GET /external-data`

### Purpose

- Returns the latest saved JSON file content

### Success Response

- `200 OK` with JSON body (stored payload + `updatedAt`)

### Error Responses

- `404 Not Found`: file does not exist yet (no successful POST yet)
- `500 Internal Server Error`: server read failure

### cURL Example

```bash
curl --location 'http://72.60.180.99/api/external-data'
```

---

## Notes for Integrators

- Every new POST fully overwrites previously stored data
- Do not send `updatedAt`; it is generated by server
- Send valid JSON only
- Recommended payload type: JSON object
